# 企业级功能路线图

本文档记录 ForgePulse 从 MVP 演进到企业级工业数据平台的功能路线。

## 1. 运营闭环

已启动：

- 告警列表、详情、确认、关闭
- 告警处理轨迹 `alarm_events`
- 设备详情与设备级遥测趋势

后续增强：

- 告警派单、升级、SLA 计时
- 班组、负责人、处理状态看板
- 维护工单和告警关联
- 事件复盘报告

## 2. 权限与审计

目标能力：

- 用户、角色、权限
- 操作审计日志
- API token 与服务账号
- 前端菜单和按钮级权限

建议阶段：

1. 先增加本地 demo 用户和 `operator`、`engineer`、`admin` 三类角色。
2. 后端引入请求上下文，所有写操作记录操作者。
3. 前端根据角色显示不同动作按钮。
4. 再接入 OIDC、LDAP 或企业 SSO。

## 3. 规则引擎

当前 worker 内置温度和压力阈值规则。企业级应拆为：

- 规则定义表
- 规则启停与版本
- 阈值、窗口、持续时间、严重等级配置
- 规则执行记录
- 告警去重、抑制、合并

## 4. 数据治理

目标能力：

- 统一数据字典
- metric 单位、量纲、上下限
- 数据质量检查
- 导入任务记录
- 数据血缘和来源说明

与公开数据集相关的导入脚本应记录：

- 数据集名称
- 下载地址
- 许可证或使用条件
- 导入时间
- 导入行数
- 字段映射版本

## 5. 可观测性

目标能力：

- 服务健康检查
- worker 消费延迟
- MQTT 消息吞吐
- API 延迟
- 数据库写入速率
- 错误日志聚合

建议先在本地实现 `/api/system/overview`，聚合服务状态和关键计数。

## 6. 部署与环境

当前以 Docker Compose 为主。后续企业部署建议：

- 拆分 dev、demo、prod 配置
- 使用 Alembic 管理数据库迁移
- 使用 CI 执行后端编译、前端构建和容器构建
- 增加镜像标签策略
- 准备 Kubernetes Helm chart 或 Kustomize

## 7. AI 能力

AI 层建议作为辅助能力，不耦合核心遥测链路。

优先顺序：

1. 工业知识库文档导入
2. 告警解释和处理建议
3. 良率异常摘要
4. 设备历史相似故障检索
5. 面向工程师的自然语言查询
